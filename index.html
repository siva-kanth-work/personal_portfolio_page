RAG Chatbot

Live

248 policy documents. Instant answers. Every claim cited.

Built for a policy think tank drowning in PDFs. Researchers spent hours hunting for answers across 248 documents, now they get cited responses in seconds. The system combines semantic search with keyword matching, reranks results for relevance, and streams answers with clickable page references. Admins can swap LLM providers, manage users, and track every query's cost and latency, all without touching code.

Inline citations [1][2] link directly to source pages in the PDF viewer
Chat interface with 247 documents, category filters, and conversation history
Admins upload PDFs; system auto-generates embeddings and enriches metadata
Every query logged: retrieval time, LLM latency, token count, cost per query
Role-based access control: Basic, Admin, Super Admin with granular permissions
Full audit trail: logins, model changes, failed attempts, config updates
Configure LLM provider, model, temperature, reasoning effort, and token pricing

Key Features

  • Hybrid retrieval: semantic (12 chunks) + BM25 keyword (5 chunks) with Cohere reranking
  • Two-model architecture: fast query analyzer + quality answer model, switchable on-the-fly
  • Clickable citations: every claim linked to source document + page number
  • Full observability: query logs, token tracking, latency metrics, cost per query
  • Enterprise-ready: 3-tier RBAC, audit logs, SSE streaming, dark mode

Self-Hosted AI Platform

Demo

All AI runs locally. Your data never leaves.

For organizations that can't send data to OpenAI or Google: healthcare, legal, finance. Every AI model runs on local hardware: Llama for conversations, Florence-2 for image captions, MiniLM for search embeddings, Z-Image for generation. Upload a photo and it's auto-captioned. Search in natural language and get instant results with AI context streaming alongside. Zero cloud dependencies, predictable costs, air-gap ready.
